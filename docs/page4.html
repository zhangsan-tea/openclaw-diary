多 Agent 分工

多 Agent 的两种思路

横向分工（按专业角色）

不同专业的事交给不同的 Agent，对话不混、文件不混、记忆不混。

纵向分工（按流程环节）

同一个任务，有负责开发的、有负责监理的、有负责审核的。

混在一起 分开
好处 信息集中，容易上下文共享 负责清晰，不互相干扰
坏处 久了容易角色混淆，身份混乱 需要主动协调，跨 Agent 信息传递

目前倾向分开为主，但信息偶尔共享时用明文传递，避免黑箱。

Agent 角色
弗洛一德 总指挥 / 养虾助理（承接模糊类需求）
写作助手 内容创作与润色；沉淀规则到企业文档，支持查看和调整，根据新信息自动迭代
数字幕僚 工作类信息的整理与分析
健康顾问 健康评估与康复指导
觉醒教练 心理学 + 正念禅修 + ICF-MCC 教练；帮助发现潜能、盲区、模式，探索更多可能
进化官 能力传承与技能推广；拥有最高权限，可读取和修改其他助手的设置
主进程 系统主进程

Agent 使用要点

觉醒教练

  • 清晰的提示词和角色设定：设定为有心理学和正念禅修背景的 ICF MCC 教练，帮助发现潜能、盲区、模式，探索更多可能。这个设定是基础，决定了对话的深度和方向。
  • 足够的上下文和背景信息：主动提供个人相关情况，包括个人经历、对话记录等，帮助它往深处思考，而不是停留在表面。
  • 日常互动积累默契：通过持续对话，慢慢觉得它很懂你，你也越来越知道怎么用它——它对你的情况、特点、模式、盲区以及变化会越来越敏锐。这个在内观类应用上非常明显。
  • 与知识库结合：复杂文档可以放到知识库里，利用龙虾的整体分析能力来看情况；也可以利用定时能力，定时拉取新信息（如来自 flomo 的随手记），跟踪新议题和情况的变化。

健康顾问

  • 输入体检报告、健康问卷等，建立个人健康档案
  • 设置定时体检/检查任务、运动康复计划、服药提醒
  • 日常健康问题互动，让其记住关键信息
  • 与苹果健康打通，分析睡眠、心跳等指标（本周发现：睡眠与心跳某关联，待查）
  • 定期扫描一周健康 APP 数据，给出本周发现和提醒

进化官

  • 整个系统的"能力传导器"，拥有最高权限
  • 具体做三件事：能力学习（负责升级系统、学习新技能）、技能传导（一个 Agent 学会的通用能力向其他 Agent 推广）、故障诊断（其他 Agent 出问题后帮诊断和修复）
  • 案例：某 Agent 学到了处理某类文件的能力 → 进化官验证可通用 → 推广到所有需要的 Agent
  • 明文优先：Agent 之间通过明文传递信息，不搞暗箱操作，避免变成自己都看不懂的系统

写作助手

  • 文采不是最重要的，对内容的理解才是
  • 写养虾心得，先跟总指挥聊；写觉醒修炼的心得，先在觉醒教练 Agent 里推进；需要打磨润色了，才找写作助手
  • 专业分工，最后才打磨
  • 写作助手负责沉淀规则，这些规则存到企业文档中，可以随时查看和调整，并根据新信息进来自动迭代
