1. 为什么要养龙虾？（一句话结论）

常规 AI 更像一个入口或工具，龙虾更像一个人。

  • 电脑权限：不只是聊天，而是真的能接触电脑和外部工具（浏览器、硬盘、网络、API）。
  • 记忆系统：拥有长期记忆、任务清单和上下文追踪。能逐步理解你，积累经验。
  • 持续执行：支持定时任务、自动巡检、主动提醒，不用每次重新下指令。
  • 文件交互：可进行本地文件批量读取、分析与整理。
  • 远程操控：通过企微/Telegram沟通，方便确立多身份 Agent 矩阵。

4. 工具选型与部署

总原则：如果有省事的办法，就不要用复杂的办法；体验是探索，集中才是生产力。

部署环境 代表产品 核心特点
外网/本地 开源 OpenClaw / EasyClaw 数据本地可控，可深度定制，权限全开放；一键版省心但模型锁死。
内网 Box (内网沙盒) 能连通外网和内网；可方便检索浏览内网知识库（IWiki/KM）；浏览器操控能力强。
内网 Workbuddy Codebuddy 的龙虾版，AICoding 能力强，权限隔离好；涉及敏感信息优先推荐。

⚠️ 云端 vs 本地的权衡：

云端产品（内网/一键版）有安全保障且免去配置烦恼；但代价是无法享受本地安装那种对权限随心所欲的修改和调整。

5. 模型选择与 Token 优化战

基本原则：需求简单随便用，复杂任务用最好的。模型高一寸，工程胜一丈。

  • 理清再问：先用便宜对话框把需求理清，再抛给龙虾，切忌用龙虾做零散问答（调取 Memory 极耗 Token）。
  • 定时用量审计：监控各助手消耗，发现异常直接问它原因。
  • 定期工作区大扫除：根目录只留核心文件（SOUL/IDENTITY），历史草稿移入 archive/，避免每次起步多耗几万 Token。
  • /new 的三步法：长会话启用 /new 前，先让系统生成 300 字精简摘要，/new 之后重载摘要，无缝续接任务。

6. 龙虾安全防线

把 AI 当员工，就必须设定红线。

  • 风险评估先行：重要操作前先评估风险度和必要性。低风险高必要性立即执行；高风险低必要性寻找替代。
  • 物理隔离密钥：填写第三方 API 密钥时，在强隔离无记忆环境（如 Workbuddy）操作，防止残留在日常 MEMORY.md 中。
  • ClawScan 体检：给 OpenClaw 配一个安全体检助手，定期扫描漏洞和越权行为。
  • 手搓自研 Skill 优先：尽量避免安装来源不明的 MCP 插件。目前系统核心依赖的 ppt-to-text、health-automation 等均为自建手搓，安全可控。

7. 实战方法论汇总

使用 AI 需要有“往后退一步”的意识。很多时候不是做不到，而是想不到。
  • 方向不清时退一步：写文章没想明白路径时，先问 AI 出个框架建议，再多轮推敲。
  • 不知怎么提问时退一步：倒回去问“我该怎么提问？”，或者先讲大概情况问“你还需要我提供什么？”。
  • 不要用固有想法套住模型：把自己发现的规律先藏着，让 AI 先分析特征，有新发现再叠加自己的想法。
  • 敢于授权：意图比流程重要。讲清楚要什么，直接对它说“我授权你，你自己想办法”。
  • 把结构壳当成能力：龙虾外面的结构壳（Harness）本身就是极强的能力扩展。

8. 多 Agent 架构与知识管理

现实观察：横向专业分工很难绝对明确。比如工作思考常跟心理状态有关（跨数字幕僚和觉醒教练）。边界模糊是常态，偶尔共享时使用明文传递避免黑箱。

知识 vs 记忆 说明与策略
知识 (可复用/结构化) 优先使用 MD 格式保存。正在探索大批量文档处理与转化。
记忆 (情境化/私人化) 随互动积累。重要信息必须对助手明确指令：“存到 Memory 里”。

语音输入谱系（精确度递减）：豆包输入法 → 微信输入法 → SaySo → Typeless。原则是不要在不必要的事情上消耗心力，模糊输入通常就足够让 AI 理解意图。
