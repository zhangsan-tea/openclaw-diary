知识管理实践

知识 vs 记忆

  • 知识：可复用、结构化、跨会话传递的公共信息
  • 记忆：个人化、情境化、随互动积累的私人信息

两者协同，但用途不同。知识用来回答问题，记忆用来理解"你是谁"。规律：知识是"知道"，记忆是"认识"。

输入格式

  • 优先 MD 格式保存知识，结构清晰，易于检索
  • 可通过 Skill 将 MD 转成讲述者模式（正在推敲中）
  • 重要信息要明确说"存到 Memory 里"或"形成 skill"，养成习惯

知识的类型与权重

类型 特点 执行置信度
结构性/原理性信息 规律清晰，可直接指导行动 高
参考性资料 需要结合情境判断，不能直接套用 低，需标注来源
个人经验总结 因人而异，需要验证 中

平台 与龙虾的结合状态 说明

OpenClaw 与知识库打通

平台 作用 特点
KNOT 内网知识库 直接调用内网信息，安全性好；与企微结合方便
IWiki / KM 内网文档检索 查内网资料方便
Box 内网信息读取 + 外网连通 非知识库定位，主要用于读取；可连通内外网；浏览器操控能力强

多平台打通是趋势，内外网信息协同是长期目标。

海量信息的批量处理

通过 Codebuddy / Workbuddy 读取本地文件夹：

  • 批量去除重复文件
  • 批量给文件重命名（文件名加创建时间）
  • 给出文件分类建议
  • 文字信息转 MD 格式（单文件和批量文件夹）
  • PPT 视觉化理解与转述（这个需求非常大，正探索中）

这类 Skill 需求量很大，正在探索中。
