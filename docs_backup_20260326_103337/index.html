一、工具选用策略与产品演进

1.1 核心决策维度

安全性优先：数据隐私保护、权限控制是首要考虑因素

权限开放度：可探索空间、自定义能力决定长期使用价值

多Agent设置便捷性：角色隔离、独立记忆支持团队协作模式

内外网连通性：内网版可访问内部资源，外网版需穿透方案

成本控制：Token费用、模型调用成本需纳入决策考量

1.2 产品演进历程与定位

  • 开源版：按教程安装，稳定性差，严重依赖Codebuddy诊断修复
  • EasyClaw：一键安装解决配置和稳定性问题，提供专家库和模型接入，但无法接入外部模型
  • OpenClaw：灵活定制，多模型支持，丰富的工具生态，适合深度用户
  • Box：具备内网连通、安全沙盒、浏览器控制等功能，本质是运行在本机的OpenClaw程序，非纯内网版也非云端，目前无法便捷配置多Agent

二、聊天软件对比与选择

2.1 Telegram（主力平台）

  • ✅ 优点：机器人创建方便、单点沟通顺畅、安全性好、已解决网络问题
  • ❌ 缺点：需要特殊网络环境（已解决）

2.2 飞书（协作场景）

  • ✅ 优点：与文档/日历深度集成、拉群协作方便
  • ❌ 缺点：权限配置繁琐、消息发送/附件读取不稳定

2.3 企业微信（内外网协同）

  • ✅ 优点：内外网打通、安全可控、与内网机器人协作
  • 🎯 现状：已实现内外网机器人协作，通过@引用实现信息互通
  • 🔍 待探索：群消息读取、机器人互@、自动协作规则

2.4 其他平台

  • Discord：同事已验证互@功能，10秒延迟规则有趣，适合团队协作

三、多助手分工策略体系

3.1 横向分工（角色专业化）

  • 觉醒教练：心理学+禅修+ICF-MCC大师，需充足上下文（咨询记录、梦境、日常思考）
  • 健康顾问：健康评估与康复指导，有明确医疗边界
  • 写作助手：内容创作与润色，专注文字表达
  • 弗洛一德：总体协调、系统运维、决策支持
  • 核心原则：给足上下文、持续对话建立默契、定时知识库归档

3.2 纵向分工（流程协同）

  • 进化官：能力传承、通用技能推广（如zip解压、工具安装）
  • 代码相关：工程开发、代码监理、甲方模拟审核
  • 通信方式：倾向明文对话，避免黑箱操作，保持可追溯性

3.3 多助理设置方法论

  • 启动阶段：最小可行团队原则，先配置1-2个核心助理，通过实际使用发现真实需求再扩展
  • 角色定义：行为边界 > 功能列表，重点定义"什么不该做"而非"能做什么"
  • 协作机制：显性优于隐性，助理间协作必须通过明确对话触发
  • 迭代优化：从错误中学习，记录角色混淆情况，定期审视职责重叠或空白

四、安全实践与风险控制

4.1 输入脱敏原则

  • 回避敏感信息：Token、密码、个人身份信息绝不直接输入
  • 内容抽象化：将具体细节提炼为规律性描述，保护隐私

4.2 工具安全管理

  • 来源验证：避免安装来源不明的Scale（技能）
  • 授权谨慎：谨慎授权个人信息，推荐云端部署而非本地电脑
  • 权限分层：建立三层权限架构（进化官最高权限→各助理专业权限→弗洛一德协调权限）

4.3 操作决策原则

  • 风险评估先行：重要系统操作前，先评估风险度和必要性
  • 执行策略：低风险高必要性→立即执行；高风险低必要性→暂缓或寻找替代方案
  • 容错机制：配置变更前备份、重要功能保留回退版本、系统故障时降级运行

六、知识库体系与信息管理

6.1 多平台并行策略

  • 内网平台：KNOT、IWiki、KM用于企业内部知识
  • 外网平台：Obsidian、NotebookLM用于个人知识管理
  • 本地存储：OpenClaw自带知识库处理临时和敏感内容

6.2 输入格式标准化

  • 优先MD格式：结构清晰，跨平台兼容性好
  • 批量处理流程：PPT/文档 → 视觉理解 → MD转换 → 知识库存储
  • 元数据标记：为不同来源内容添加标签，便于后续检索

6.3 知识 vs 记忆分层

  • 知识：可复用的结构化信息（操作手册、技术规范）
  • 记忆：个人化的上下文信息（项目背景、个人偏好）
  • 三层记忆体系：短期会话上下文 + 长期MEMORY.md + 结构化知识库

6.4 内外网协同方案

  • 信息获取：内网信息脱敏后通过企微接口间接获取
  • 协作机制：外网助理通过@引用方式调用内网机器人
  • 安全把关：人在其中做最终安全把关，AI做前置过滤

七、使用心得与最佳实践

7.1 心态建设

  • "养虾一定会挂掉"：接受故障是常态，不因挫折而放弃
  • "把AI当员工"：明确指令、充分授权、结果导向的管理思维

7.2 实用技巧

  • 明确存储：重要信息明确要求"存到md里"或"形成scale"
  • 大胆授权：面对权限不足，直接说"我授权你做，自己想办法"
  • 能力复用：通过进化官推广通用技能，实现"一次学习，全系统受益"

7.3 维护策略

  • 预防性管理：定期备份关键配置文件，建立版本回退机制
  • 主动优化：HEARTBEAT机制承担记忆维护、技能审视等主动优化任务
  • 监控诊断：进化官主动监控 + Codebuddy底层修复，形成完整闭环

八、系统架构与运维管理

8.1 权限与安全边界

  • 三层架构：进化官（最高权限）→ 各助理（专业权限）→ 弗洛一德（协调权限）
  • 任务分发：总指挥分解 → 专业助理执行 → 进化官验证的流水线模式
  • 避免混乱：复杂任务采用明确分工，避免单助理处理时的角色混乱

8.2 版本与配置管理

  • Git同步：内网Git仓库管理配置文件，确保多环境一致性
  • 心跳检查：HEARTBEAT定期检查配置同步状态和系统健康
  • 配置备份：重要变更前先在测试环境中验证

8.3 扩展能力生态

  • MCP工具集成：将本地Mac能力（提醒事项等）封装为MCP工具
  • 跨平台协同：通过企微机器人实现内外网信息互通
  • Skills管理：重复性解决方案抽象为可复用Skills，通过进化官推广
